# Virunga Performance Dashboard - Confidential Case Study

## Overview

This project represents a dashboard product developed for KPI tracking, reporting visibility, and stakeholder monitoring in a professional environment where the live data cannot be shared publicly.

The public portfolio version focuses on the reporting approach, dashboard logic, and business value rather than the protected dataset.

## Problem

Stakeholders needed a more centralized way to track performance signals and follow operational or program indicators without switching between disconnected files and manual summaries.

## Solution

The dashboard was designed to improve visibility by bringing selected indicators into a structured reporting interface with clearer navigation, summarized metrics, and more consistent monitoring views.

Key contributions that can be described publicly:
- dashboard-oriented KPI presentation
- reporting structure for faster stakeholder review
- improved readability of performance information
- interface support for ongoing monitoring and updates

## What this demonstrates

- KPI dashboard design
- reporting for non-technical stakeholders
- translating complex information into usable views
- experience with real-world dashboard workflows beyond mock datasets
- discretion and professionalism with confidential systems
